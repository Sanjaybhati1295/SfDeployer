Overview

01

SF Deploy Manager is a Chrome extension that lets Salesforce developers and admins deploy metadata components between orgs directly from their browser — without needing complex setups or external tools.

We are fully committed to your privacy. This policy explains exactly what data is accessed, why it's needed, and where it lives.

✅

Everything stays on your device. SF Deploy Manager does not run any servers, does not collect usage data, and only communicates with Salesforce orgs that you explicitly connect to.

🗂️

Data We Collect

02

The extension only stores what is strictly needed for it to work. Here's a complete breakdown:

Data Type Why It's Needed Sent Externally?
Org aliases & access tokens Authenticate with your Salesforce orgs Never
Deployment history Track past deployments & enable rollback Never
Metadata snapshots Restore components to previous state Never
User preferences Remember your UI settings & defaults Never
Active tab URL Auto-detect current Salesforce org Never
⚠️

We do not collect names, emails, IP addresses, usage analytics, crash reports, or any personally identifiable information.

💾

How Data is Stored

03

All data is stored locally on your machine using Chrome's built-in storage APIs. Nothing ever touches an external server owned by us.

WhereWhat's Stored ThereLocation
Chrome local storage Org connections, preferences, deployment logs Your Device
Chrome unlimitedStorage Deployment snapshots & metadata files Your Device
Salesforce API calls Metadata goes directly to your own org — never to our servers Your Org Only

🗑️ Clear all data anytime: Chrome Settings → Extensions → SF Deploy Manager → Clear Data

🔑

Permissions Explained

04

We only request permissions that are absolutely essential. Here's exactly why each one is needed — in plain English:

  • storage Saves org connections, deployment history, and settings locally on your device. No data leaves your machine.
  • unlimitedStorage Metadata snapshots for rollback can be large. Chrome's standard 5MB quota isn't enough — this lets us store full deployment history and rollback files locally.
  • tabs Reads the URL of your active tab to auto-detect which Salesforce org you're on — so you don't have to select it manually every time.
  • host permissions Required to communicate with Salesforce APIs (*.salesforce.com, *.force.com) to retrieve and deploy metadata to your own orgs.
ℹ️

If a future update requires a new permission, we will notify you and explain why before you accept.

☁️

Salesforce Authentication

05

We use Salesforce's standard OAuth 2.0 web flow — the same secure method used by VS Code and Salesforce CLI.

  • OAuth 2.0 You log in directly on Salesforce's official login page. Your password is never seen, intercepted, or stored by this extension.
  • Access Tokens OAuth tokens are stored securely in Chrome's local storage and used only to make API calls to your own Salesforce org.
  • Direct API Calls All Salesforce requests go directly from your browser to your org. No traffic is routed through any server we operate.
🔐

Revoke access anytime: Salesforce Setup → Connected Apps OAuth Usage
