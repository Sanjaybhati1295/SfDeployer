Overview

SF Deploy Manager ("the Extension") is a Chrome browser extension that enables Salesforce developers and administrators to deploy metadata components between Salesforce orgs directly from their browser.

We are committed to protecting your privacy. This policy explains exactly what data the extension accesses, how it is used, and where it is stored. The short answer: everything stays on your device — nothing is transmitted to our servers.

✅

Bottom line: SF Deploy Manager does not operate any servers, does not collect telemetry, and does not have access to your Salesforce data outside of what you explicitly initiate through the extension UI.

🗂️

Data We Collect

The extension only stores data that is strictly necessary for its core functionality. All data is stored locally on your device.

Data Type Purpose Sent Externally?
Org aliases & access tokens Authenticate with connected Salesforce orgs Never
Deployment history Track past deployments for audit & rollback Never
Metadata snapshots Enable rollback to previous component states Never
User preferences Remember UI settings and defaults Never
Active tab URL Auto-detect current Salesforce org Never
⚠️

We do not collect names, email addresses, IP addresses, usage analytics, crash reports, or any other personally identifiable information.

💾

How Data is Stored

All data generated by the extension is stored exclusively on your local machine using Chrome's built-in storage APIs and your local filesystem:

Storage Location What's Stored
Chrome local storage Org connections, preferences, deployment metadata
Chrome unlimitedStorage Deployment snapshots and metadata component files
Salesforce servers only Metadata API calls go directly to your own Salesforce org — never to our servers

You can clear all locally stored data at any time via Chrome Settings → Extensions → SF Deploy Manager → Clear Data.

🔑

Permissions Explained

The extension requests only the minimum permissions necessary. Here is why each permission is required:

  • storage Saves org connections, deployment history, and user preferences locally on your device. No data leaves your machine.
  • unlimitedStorage Metadata snapshots used for rollback can be large. Standard Chrome storage quota (5MB) is insufficient — this permission allows adequate local storage for full deployment history.
  • tabs Reads the URL of the active tab to auto-detect which Salesforce org you are currently viewing — eliminating the need for manual org selection.
  • host permissions
    (*.salesforce.com, *.force.com)     Required to communicate with Salesforce APIs (Metadata API, Tooling API) to retrieve and deploy metadata components to your own Salesforce orgs.
☁️

Salesforce Authentication

The extension uses Salesforce's standard OAuth 2.0 web flow for authentication. This means:

  • OAuth 2.0 You log in directly on Salesforce's official login page. Your Salesforce password is never seen or stored by this extension.
  • Access Tokens Tokens are stored locally using Chrome's secure storage API and are used only to make API calls to your own Salesforce org.
  • API Calls All Salesforce API requests go directly from your browser to your Salesforce org. No requests are routed through any intermediate server we operate.
🔐

You can revoke access at any time from your Salesforce org under Setup → Connected Apps OAuth Usage.

🔗

Third-Party Services

SF Deploy Manager does not integrate with or transmit data to any third-party services, analytics platforms, advertising networks, or cloud storage providers.

The only external communication the extension performs is direct API calls to your own Salesforce org instances (e.g., yourorg.salesforce.com).
